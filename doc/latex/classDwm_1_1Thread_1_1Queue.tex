\doxysubsection{Detailed Description}
\subsubsection*{template$<$typename \+\_\+\+Value\+Type$>$\newline
class Dwm\+::\+Thread\+::\+Queue$<$ \+\_\+\+Value\+Type $>$}
This template provides inter-\/thread first-\/in first-\/out (FIFO) queueing. 

A source thread may insert objects into the back of the queue using \doxylink{classDwm_1_1Thread_1_1Queue_ab299aa8a99b55ca3d8060318b3929cd3}{Push\+Back()} while a sink thread pops objects from the front of the queue using \doxylink{classDwm_1_1Thread_1_1Queue_ab2e201afc119ed280ad25760b65ffedb}{Pop\+Front()}. A sink thread may use \doxylink{classDwm_1_1Thread_1_1Queue_a67e9203f20d26df3ed6f8722aff22eb1}{Wait\+For\+Not\+Empty()} to wait for entries to be added to the queue; the calling thread will be blocked until the queue is non-\/empty. A sink thread may also choose to use \doxylink{classDwm_1_1Thread_1_1Queue_a69b401fa39361443175d566b0c738319}{Condition\+Wait()} and \doxylink{classDwm_1_1Thread_1_1Queue_ab2e201afc119ed280ad25760b65ffedb}{Pop\+Front()}, which allows a source to wake up the sink using Condition\+Broadcast() or \doxylink{classDwm_1_1Thread_1_1Queue_af1576afd114e7acc61cc7bed18f9471e}{Condition\+Signal()} without pushing an entry into the queue.
