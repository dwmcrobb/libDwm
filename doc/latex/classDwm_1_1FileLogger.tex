\doxysubsection{Detailed Description}
This class encapsulates syslog-\/like logging, but uses files instead of the syslog facility. 

You may set the maximum size of the log file (in bytes) by using the \doxylink{classDwm_1_1FileLogger_ad6c7a0c5df881f1d492bb55f8ca5b0d3}{Rollover\+Size(uint32\+\_\+t)} member. The default is 102400 (100K) bytes. You may set the number of rollover files using the \doxylink{classDwm_1_1FileLogger_a84258b4103269d6b8c2968c49fd9e1da}{Num\+Rollover\+Files(uint32\+\_\+t)} member. The default is 1. The logger will roll over "{}logfile"{} to "{}logfile.\+0"{} and "{}logfile.\+N"{} to "{}logfile.\+N+1"{} for \{ 0 \texorpdfstring{$<$}{<}= N \texorpdfstring{$<$}{<} \doxylink{classDwm_1_1FileLogger_ad5f80296d09529180caa10b3ebc09537}{Num\+Rollover\+Files()} \}. If the number of rollover files is set to 0, the log file will be truncated each time it grows beyond \doxylink{classDwm_1_1FileLogger_a7fc66f8dee3d7212c29df561f49ab8c7}{Rollover\+Size()}. Note that \doxylink{classDwm_1_1FileLogger_a7fc66f8dee3d7212c29df561f49ab8c7}{Rollover\+Size()} is only an estimate; the actual log size will be slightly larger since we treat log messages as atomic (we don\textquotesingle{}t split them across files). You may set a priority threshold with \doxylink{classDwm_1_1FileLogger_a5a56eafccec65f72efe3fbcbfd51f1cb}{Minimum\+Priority(int)}; any messages below this priority will be silently discarded. This allows dynamic logging levels, as well as allowing syslog.\+conf-\/like priority filtering functionality.
