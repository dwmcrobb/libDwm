Opens the logger for syslog logging. 

{\ttfamily ident}, {\ttfamily logopt} and {\ttfamily facility} are used directly to call openlog() (see the syslog(3) manpage). Returns true on success, false on failure. Note that openlog() gives no indication of failure, hence we only return false if we get an unacceptable {\ttfamily logopt} or {\ttfamily facility}. The acceptable facilities are {\ttfamily LOG\+\_\+\+USER}, {\ttfamily LOG\+\_\+\+DAEMON}, {\ttfamily LOG\+\_\+\+AUTH}, {\ttfamily LOG\+\_\+\+LOCAL0}, {\ttfamily LOG\+\_\+\+LOCAL1}, {\ttfamily LOG\+\_\+\+LOCAL2}, {\ttfamily LOG\+\_\+\+LOCAL3}, {\ttfamily LOG\+\_\+\+LOCAL4}, {\ttfamily LOG\+\_\+\+LOCAL5}, {\ttfamily LOG\+\_\+\+LOCAL6}, and {\ttfamily LOG\+\_\+\+LOCAL7}. The acceptable log options are {\ttfamily LOG\+\_\+\+PID}, {\ttfamily LOG\+\_\+\+CONS} and {\ttfamily LOG\+\_\+\+NDELAY} on all systems. {\ttfamily LOG\+\_\+\+PERROR} is accepted on systems which support it. {\ttfamily LOG\+\_\+\+NOWAIT} is accepted on systems which support it (Solaris, for example). Note this member can cause adverse affects on external syslog() calls because syslog facilities are global.
