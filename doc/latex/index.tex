\chapter{Dwm Class Library}
\hypertarget{index}{}\label{index}\index{Dwm Class Library@{Dwm Class Library}}
\hypertarget{index_intro_sec}{}\doxysection{\texorpdfstring{Introduction}{Introduction}}\label{index_intro_sec}
This class library is just a (somewhat random) collection of classes I\textquotesingle{}ve developed over the years for various applications. Since these are used in a variety of places, they\textquotesingle{}re collected here to make re-\/use easy.

There are a few classes herein that I\textquotesingle{}ve used on nearly every software project in the last 20 years. One of those would be the \doxylink{classDwm_1_1SysLogger}{Dwm\+::\+Sys\+Logger} class with the \doxylink{DwmSysLogger_8hh_a7fd0cf2c423c73ded4f5e1332b570ea7}{Syslog()} macro, since syslogging is a common need. Another would be the \doxylink{classDwm_1_1Assertions}{Dwm\+::\+Assertions} class with the \doxylink{DwmUnitAssert_8hh_ac6ef8a61e8418d80847517047c2c8958}{Unit\+Assert()} macro since I almost always write unit tests as part of my development process. And when I need to store data, I use the IO classes along with the various Readable and Writable interfaces.

Since much of the software I\textquotesingle{}ve written in the last 20 years has been multithreaded, I have also made heavy use of the \doxylink{classDwm_1_1Thread_1_1Queue}{Dwm\+::\+Thread\+::\+Queue} class for inter-\/thread communication (usually as \textquotesingle{}work queues\textquotesingle{}).\hypertarget{index_history_sec}{}\doxysection{\texorpdfstring{History}{History}}\label{index_history_sec}
I started this library in 1998. I continue to maintain it for my personal use. While it has grown over the years, the most common use features have been stable for a long time. To some extent they have to be stable; I have over 500,000 lines of code in my personal software repository, and a decent amount of it is dependent on this library. The good news is that those dependencies don\textquotesingle{}t prevent me from refactoring, improving and growing lib\+Dwm as needed.\hypertarget{index_platforms_sec}{}\doxysection{\texorpdfstring{Platforms}{Platforms}}\label{index_platforms_sec}
I really only maintain support for 4 platforms\+: Free\+BSD, mac\+OS, desktop linux and Raspbian. Free\+BSD is my operating system of choice for servers and mac\+OS is my operating system of choice for desktops and laptops. I have several Raspberry Pis I utilize for simple small tasks, and Ubuntu VMs and Ubuntu workstations.\hypertarget{index_smattering_sec}{}\doxysection{\texorpdfstring{A small sample of what\textquotesingle{}s in lib\+Dwm}{A small sample of what\textquotesingle{}s in lib\+Dwm}}\label{index_smattering_sec}
\hypertarget{index_dwm_io_sec}{}\doxysubsection{\texorpdfstring{I/O -\/ Serialization}{I/O -\/ Serialization}}\label{index_dwm_io_sec}
\doxylink{classDwm_1_1DescriptorIO}{Dwm\+::\+Descriptor\+IO}, \doxylink{classDwm_1_1FileIO}{Dwm\+::\+File\+IO}, \doxylink{classDwm_1_1StreamIO}{Dwm\+::\+Stream\+IO}, \doxylink{classDwm_1_1ASIO}{Dwm\+::\+ASIO}, \doxylink{classDwm_1_1GZIO}{Dwm\+::\+GZIO} and \doxylink{classDwm_1_1BZ2IO}{Dwm\+::\+BZ2\+IO} provide serialization for intrinsic types, containers of intrinsic types, user types which implement the required interfaces, and containers of these user types.

Intrinsic integer types are written in network byte order. The library takes care of byte ordering so you need not be concerned with it. Float and double types are written in IEEE 754-\/1985 form (see RFC 1832). Some types are stored as (length,value) pairs, including the standard containers and C++ string objects.

For user types, the required interfaces are \texorpdfstring{$\ast$}{*}\+Readable and \texorpdfstring{$\ast$}{*}\+Writable, such as \doxylink{classDwm_1_1StreamReadable}{Dwm\+::\+Stream\+Readable} and \doxylink{classDwm_1_1StreamWritable}{Dwm\+::\+Stream\+Writable}. The required interfaces for \doxylink{classDwm_1_1GZIO}{Dwm\+::\+GZIO} are \doxylink{classDwm_1_1GZReadable}{Dwm\+::\+GZReadable} and \doxylink{classDwm_1_1GZWritable}{Dwm\+::\+GZWritable}. The required interfaces for \doxylink{classDwm_1_1BZ2IO}{Dwm\+::\+BZ2\+IO} are \doxylink{classDwm_1_1BZ2Readable}{Dwm\+::\+BZ2\+Readable} and \doxylink{classDwm_1_1BZ2Writable}{Dwm\+::\+BZ2\+Writable}.

Today, the C++ programmer has several options for serialization, and the possibility of reflection in a future version of the standard. However, I wrote most of this functionality in 2004 when we didn\textquotesingle{}t have a lot of options to support serialization and finalized most of it in 2007 with the addition of cleaner code for std\+::tuple. I couldn\textquotesingle{}t do this effectively until the compilers were up to speed.
